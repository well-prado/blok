Junior · Phase 6 · Empty states Each empty state teaches. Real CLI command, real SDK snippet, real next-action button. No mascot, no "Oops! Nothing here yet". The operator should be able to act within five seconds of landing. Per skill rules: no decorative SVG illustrations — a single Lucide glyph at half-opacity is enough. Whitespace is the design, not the failure.

Empty states · four pages, one pattern

Each cell below is one of Studio's primary pages in a state where the data set is empty. The pattern: one-glyph anchor · headline · single-paragraph lede · concrete next action · escape hatch. No page resolves to "this list is empty" without a clear path forward.

Hierarchy: glyph (visual anchor, half-opacity) → headline (Newsreader italic, what state we're in) → lede (sans, plain English, names the thing missing) → action (primary green CTA, does the highest-leverage thing) → escape (secondary, undoes the active filter / reverses navigation). No friendly cartoons, no "click here to learn more" pop-ups, no checklists.

The pattern, summarized

  1. Glyph — single Lucide line-icon, 32 px, 50% opacity. Anchors the eye and signals "this is intentional space," not "the page broke."
  2. Headline — Newsreader italic (the only place we keep the Direction-C concession). Names the state in 3–6 words: "No runs yet · production", "No runs match these filters", "No matches in the live stream", "Connecting…"
  3. Lede — sans-serif, 13 px, max 420 px wide. Explains why the list is empty and what would put data into it. Names variables in code.
  4. Snippet (cells 01) — only when teaching how to start. Real curl, real SDK call, copy button. No fake hello-world; uses an actual workflow name from this codebase.
  5. Action row — primary green CTA does the highest-leverage thing for the empty state (run sample / clear filters / clear search). Secondary widens the funnel. Ghost saves the view or escapes.

What I deliberately didn't do

  • No mascot · no friendly cartoon character · no SVG-illustrated empty boxes
  • No "Pro tip:" / "Did you know?" callouts in empty space
  • No checklists ("1. Create a workflow 2. Trigger it 3. Watch the magic")
  • No giant "Get started" button as the only thing on the page
  • No fake skeleton data that stays visible — the loading skeleton (cell 04) preserves structure, not phantom content
