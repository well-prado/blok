A workflow body runs straight through by default. The **primitives** in this
section let you control flow: pick a branch, iterate a list, recover from an
error, pause and resume later.

In the `@blokjs/core` typed-handle DSL, primitives are **functions you call
inside the workflow callback** — each arm is its own callback where `step()`
calls register. They compose freely with
[idempotency](/d/reliability/idempotency), [retry](/d/reliability/retry),
[sub-workflows](/d/composition/sub-workflows), and each other.

## What a primitive looks like

```ts
import { workflow, step, branch, forEach, switchOn, tryCatch, gt, http } from "@blokjs/core";
import validate from "../nodes/validate";
import route from "../nodes/route";

export default workflow("orders.route", { version: "1.0.0", trigger: http.post("/orders") }, (req) => {
  const order = step("validate", validate, { raw: req.body });

  branch("lane", gt(order.qty, 10), {
    then: () => { step("bulk", route, { lane: "bulk" }); },
    else: () => { step("standard", route, { lane: "standard" }); },
  });
});
```

Each primitive returns a typed handle (or `void`), and each arm callback runs
in its own scope. The [handles & footguns](./handles-and-footguns.mdx) page is
the contract for what a handle is readable from across arms.

## The DSL primitives

These ship as functions in `@blokjs/core`:

| Primitive | Signature | Use it when |
|---|---|---|
| [`branch`](./branch.mdx) | `branch(id, cond, { then, else? })` | Two-way if/else routing on a typed condition |
| [`switchOn`](./switch.mdx) | `switchOn(disc, { cases, default? }, { id })` | N-way routing on a single discriminant |
| [`forEach`](./forEach.mdx) | `forEach(iterable, (item, i) => {…}, opts?)` | Iterate an array (sequential or parallel) |
| [`tryCatch`](./tryCatch.mdx) | `tryCatch(id, { try, catch, finally? })` | Recover from a failing step group |

## Object-style step shapes

Two control-flow shapes are **not** part of the typed-handle DSL — they live
only as object-style / JSON step shapes:

| Shape | Form | Use it when |
|---|---|---|
| [`loop`](./loop.mdx) | `{ until, do, maxIterations }` | Poll until a condition is met |
| [`wait`](./wait.mdx) | `{ for }` / `{ until }` | Pause a run for a duration, then resume |

For condition-based repetition reach for `loop`; for a trigger-level deferral
instead of a mid-flight pause, see [`delay` & TTL](/d/scheduling/delay-and-ttl).

## Nested primitives

Primitives nest freely. A `forEach` body can contain a `branch`; a `switchOn`
case can contain a `forEach`; a `tryCatch.try` can wrap any of them. All of
them share one flat `ctx.state` namespace — the
[handles & footguns](./handles-and-footguns.mdx) page covers which handle is
readable from across arms and the four mistakes that namespace surfaces.

## Composition with reliability

The reliability knobs ([idempotency](/d/reliability/idempotency),
[retry](/d/reliability/retry), [maxDuration](/d/reliability/max-duration)) live
on the **regular `step()` calls inside** primitives, not on the primitive
itself. Cache an LLM call inside a `forEach` body, retry an API call inside a
`tryCatch` try arm — same options object as any other step.
