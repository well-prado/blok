`branch` is `if/else` for workflows. It evaluates a condition and runs the `then` arm when
truthy, the `else` arm otherwise. Each arm is a **callback** where `step()` calls register into
that branch.

## Quickstart

```ts
import { workflow, step, branch, gt, http } from "@blokjs/core";
import validateOrder from "../nodes/validate-order";
import route from "../nodes/route";

export default workflow("orders.route", { version: "1.0.0", trigger: http.post("/orders") }, (req) => {
  const order = step("validate", validateOrder, { qty: req.body.qty });

  branch("lane", gt(order.qty, 10), {
    then: () => { step("bulk", route, { lane: "bulk" }); },
    else: () => { step("standard", route, { lane: "standard" }); },
  });
});
```

`gt(order.qty, 10)` is a **typed condition** built from a handle. The `then`/`else` callbacks
each register their own steps.

## The condition

`branch(id, condition, arms)` accepts:

- A **typed comparator** ‚Äî `eq`, `ne`, `gt`, `gte`, `lt`, `lte`, `not` ‚Äî built from a handle and
  a value (or two handles):

  ```ts
  branch("vip", gte(order.total, 1000), { then: () => { /* ‚Ä¶ */ } });
  branch("guest", eq(req.headers["x-role"], "guest"), { then: () => { /* ‚Ä¶ */ } });
  ```

- A **boolean handle** ‚Äî a truthiness check on a node output field:

  ```ts
  const stock = step("check", checkStock, { sku: order.sku });
  branch("in-stock", stock.available, {
    then: () => { step("ship", ship, { id: order.id }); },
    else: () => { step("backorder", backorder, { id: order.id }); },
  });
  ```

<Warning>
The condition must come from a comparator or a boolean handle ‚Äî **never a hand-written string**.
A raw `"ctx.state.x > 10"` or a `js/‚Ä¶` string bypasses the typed lowering and throws at runtime.
The comparators emit the correct form for you.
</Warning>

## Using a result downstream

Handles minted **inside** an arm don't escape it ‚Äî reading `step("bulk", ‚Ä¶)`'s handle after the
branch resolves to `undefined`. When both arms produce a value you need later, write them to one
shared key with `as:`:

```ts
branch("lane", gt(order.qty, 10), {
  then: () => { step("bulk", route, { lane: "bulk" }, { as: "lane" }); },
  else: () => { step("standard", route, { lane: "standard" }, { as: "lane" }); },
});

// Both arms wrote ctx.state.lane ‚Äî now readable by id "lane".
step("notify", notify, { lane: req.body.email });
```

This is footgun #1 ‚Äî see [Handles & footguns](/d/primitives/handles-and-footguns).

## Common patterns

<AccordionGroup>
  <Accordion title="Conditional side effect (no else)" icon="bolt">
    Omit `else` when one side has work and the other is a no-op:

    ```ts
    branch("alert-on-error", not(result.ok), {
      then: () => { step("alert", notify, { message: result.error }); },
    });
    ```
  </Accordion>
  <Accordion title="Guarded sub-workflow dispatch" icon="shield">
    Run an expensive child only when a predicate matches; pair with `idempotencyKey` so retries
    don't double-charge:

    ```ts
    branch("maybe-charge", req.body.requireBilling, {
      then: () => {
        step("charge", null, { amount: req.body.amount }, {
          subworkflow: "charge-card",
          idempotencyKey: req.body.requestId,
        });
      },
    });
    ```
  </Accordion>
  <Accordion title="Nested branches" icon="code-branch">
    Branch arms can contain any primitive ‚Äî `forEach`, `tryCatch`, sub-workflows, or another
    `branch`. For 3+ cases on one value, reach for [`switchOn`](/d/primitives/switch) instead of
    nesting.
  </Accordion>
</AccordionGroup>

## Footguns

- **Arm-scoped handles** don't escape their arm ‚Äî use a shared `as:` key (above).
- **Unique step ids** ‚Äî every step id is flat per workflow, including across `then`/`else`. Reusing
  one throws at load time; use `as:` to write the same key from both arms.
