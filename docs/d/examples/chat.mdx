A complete chat demo that streams tokens from an OpenAI-compatible
endpoint (OpenRouter by default) into a browser via Server-Sent Events.
Three workflows + one UI node, mounted on a single HTTP port.

Shipped under `--examples` since v0.6.7. Swap models at runtime via
`OPENROUTER_MODEL` вҖ” same workflow against GPT-4o, Claude, Gemini,
Llama, or a local Ollama instance.

## Prerequisites

- A Blok project with **HTTP + SSE** triggers
- An [OpenRouter](https://openrouter.ai/keys) API key (free tier
  covers thousands of GPT-4o-mini messages)
- Node.js v22 or later, Bun installed

## Scaffold

```bash
npx -y -p blokctl@latest blokctl create project \
  --name chatapp \
  --triggers http,sse \
  --runtimes node \
  --examples
cd chatapp
```

Open `.env.local` and set your key:

```bash
OPENROUTER_API_KEY=sk-or-v1-<your-key>
OPENROUTER_MODEL=openai/gpt-4o-mini
```

Boot:

```bash
bun run dev
```

Open `http://localhost:4000/chat` in a browser. Send a message вҖ” tokens
arrive in real time.

## How it works

The browser opens an `EventSource` to receive tokens; HTTP POSTs deliver
each new message.

[`@blokjs/llm-stream`](/d/reference/helpers/llm-stream) publishes to an
in-process event bus, and the SSE trigger pumps every event to whichever
browser is subscribed to that channel.

## The three workflows

### `src/workflows/chat-page.ts`

Renders the chat UI page. The `chat-ui` node is a UI node shipped
with `--examples` вҖ” it returns a vanilla HTML page (no React, no build
step) with an `EventSource` + `fetch` pair wired up. Import it from
`src/nodes/` like any local node:

```ts src/workflows/chat-page.ts
import { workflow, step, http } from "@blokjs/core";
import chatUi from "../nodes/chat-ui";

export default workflow(
  "chat.page",
  { version: "1.0.0", trigger: http.get("/chat", { accept: "text/html" }) },
  () => {
    step("render", chatUi, {});
  },
);
```

### `src/workflows/chat-message.ts`

POST handler. Reads the conversation history from the request body and
runs `LlmStreamNode`. The helper streams each token to the bus channel
`chat:<sessionId>` and returns the accumulated text when the model
finishes. `tpl` embeds the `sessionId` handle into the channel string:

```ts src/workflows/chat-message.ts
import { workflow, step, http, tpl } from "@blokjs/core";
import { LlmStreamNode } from "@blokjs/helpers";

export default workflow(
  "chat.message",
  {
    version: "1.0.0",
    trigger: http.post("/chat/:sessionId/message", { accept: "application/json" }),
  },
  (req) => {
    step("stream", LlmStreamNode, {
      channel: tpl`chat:${req.params.sessionId}`,
      apiKey: process.env.OPENROUTER_API_KEY,
      model: process.env.OPENROUTER_MODEL || "openai/gpt-4o-mini",
      messages: req.body.messages,
    });
  },
);
```

### `src/workflows/chat-stream.ts`

SSE subscriber. Mounts on the HTTP server's Hono app and pumps the bus
channel out as SSE frames. The browser's `EventSource` receives
`token` events as the model produces them and a final `done` event
with the full reply. The `sse` trigger is a raw block вҖ” pass it through
`opts.trigger`:

```ts src/workflows/chat-stream.ts
import { workflow, step } from "@blokjs/core";
import { SseSubscribeNode, SseStreamNode } from "@blokjs/helpers";

export default workflow(
  "chat.stream",
  {
    version: "1.0.0",
    trigger: {
      sse: {
        path: "/sse/chat/:sessionId",
        heartbeatInterval: 15_000,
        retryInterval: 3_000,
      },
    },
  },
  () => {
    const sub = step("sub", SseSubscribeNode, { channels: ["chat:{sessionId}"] });
    step("stream", SseStreamNode, { source: sub });
  },
);
```

The `{sessionId}` placeholder in `channels` is filled from the request
path parameter at run time, so every connected browser subscribes only
to its own conversation. The `sub` handle flows straight into the
`stream` step's `source` вҖ” no `ctx.state` strings.

## Customize

### Swap the model

Change `OPENROUTER_MODEL` in `.env.local` вҖ” no code change required.
OpenRouter exposes 100+ models behind one OpenAI-compatible API:

```bash
OPENROUTER_MODEL=anthropic/claude-3.5-sonnet
# or
OPENROUTER_MODEL=google/gemini-2.0-flash-001
# or
OPENROUTER_MODEL=meta-llama/llama-3.3-70b-instruct
```

### Point at OpenAI directly

Set the OpenAI API key as `OPENAI_API_KEY` and override the base URL
in the `stream` step:

```ts
step("stream", LlmStreamNode, {
  apiKey: process.env.OPENAI_API_KEY,
  baseUrl: "https://api.openai.com/v1",
  model: "gpt-4o-mini",
  messages: req.body.messages,
  channel: tpl`chat:${req.params.sessionId}`,
});
```

### Point at local Ollama

```ts
step("stream", LlmStreamNode, {
  apiKey: "ollama-local",
  baseUrl: "http://localhost:11434/v1",
  model: "llama3.2",
  messages: req.body.messages,
  channel: tpl`chat:${req.params.sessionId}`,
});
```

### Add server-side memory

The base example sends the full history with every POST вҖ” the server
is stateless. For a Redis-backed memory chain that survives reloads,
see the [Redis-memory chat example](./chat-memory.mdx).

### Tool calls

To let the model call tools (weather lookup, calculator, function
calling), use [`@blokjs/llm-agent`](./llm-agent.mdx) instead of
`llm-stream` вҖ” same SSE pattern, plus `tool_call` and `tool_result`
events.
