## About

Welcome to the Blok examples section! Here, you will find practical examples demonstrating how to use Blok to build various types of applications and implement common patterns. Each example is designed to be a hands-on guide, complete with explanations and code snippets.

## Getting Started with Examples

These examples are structured as self-contained projects that can be easily integrated into a standard Blok project created with `blokctl`.

To run these examples, you will need:

1. A Blok project set up (see [First Steps](../introduction/first-steps.mdx)).
2. The `blokctl` CLI tool.
3. Any specific dependencies mentioned in the individual example (e.g., database clients, external API keys if applicable).

Each example has its own documentation with detailed setup instructions, code explanations, and usage guidelines.

## Available Examples

Every example ships under `--examples` when you scaffold a Blok
project — `npx blokctl@latest create project --examples`. Pick a
trigger combination that matches the example you want, and the
workflows + helper nodes are copied into your project ready to run.

### Foundational

* **[Countries API](./countries.mdx)** — fetch data from an external API
  with `@blokjs/api-call` and return it as JSON. The simplest possible
  v2 workflow. *Needs: HTTP trigger.*

* **[Database Smart Query (DB Manager)](./db-manager.mdx)** — AI-powered
  natural-language to SQL UI backed by PostgreSQL. Demonstrates UI
  nodes, branch routing, and custom nodes wrapping a database client.
  *Needs: HTTP trigger + Postgres + an OpenAI API key.*

### LLM + streaming (v0.6.7+)

* **[Streaming chat](./chat.mdx)** — token-by-token LLM streaming from
  any OpenAI-compatible model. Uses the in-process SSE bus and the
  parameterized `chat-ui` node. Swap models via env var.
  *Needs: HTTP + SSE triggers + OpenRouter (or any OpenAI-compat) key.*

* **[Chat with Redis-memory](./chat-memory.mdx)** — server-side
  conversation history. The LLM remembers prior turns across browser
  reloads. Showcases `@blokjs/redis-kv` + multi-step state composition.
  *Needs: HTTP + SSE triggers + Redis + OpenRouter key.*

* **[LLM agent with tool calls](./llm-agent.mdx)** — OpenAI function
  calling loop dispatched to HTTP endpoints. The new
  `@blokjs/llm-agent` helper handles the loop, your tools are sibling
  workflows or any URL.
  *Needs: HTTP + SSE triggers + Redis + OpenRouter key.*

### Integration patterns (v0.6.10+)

* **[Webhook router (Stripe + GitHub + Linear)](./webhook-router.mdx)**
  — provider-signed webhook receivers with HMAC verification and
  polymorphic sub-workflow dispatch. The custom-signature config
  covers any HMAC-signed provider.
  *Needs: HTTP + webhook triggers + provider signing secrets.*

* **[Worker fan-out with NATS](./worker-fanout.mdx)** — async parallel
  job dispatch with per-tenant `concurrencyKey` + per-job
  `idempotencyKey` + retry. Demonstrates the worker trigger end-to-end.
  *Needs: HTTP + worker triggers + a broker (NATS / Redis / BullMQ /
  SQS / Kafka).*

## How to Use This Section

1. **Choose an example**: Select either the simple Countries API example or the more advanced DB Manager example based on your needs.
2. **Read the overview**: Each example page starts with a description of its purpose and what it demonstrates.
3. **Examine the code**: Review the Node implementations and the Workflow definitions provided in the documentation.
4. **Follow setup instructions**: Make sure to follow the specific setup steps (e.g., installing dependencies, setting environment variables).
5. **Run the example**: Execute the workflow as described, usually by starting the Blok runner and accessing it via a browser or API client.
6. **Experiment**: Modify the example code to better understand its workings or to adapt it to your own needs.

These examples are intended to be a starting point. We encourage you to build upon them and explore the full potential of Blok .

If you have an idea for a new example or want to contribute, please check our [Contributing Guide](/c/overview/how-to-contribute).
