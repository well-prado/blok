This example builds a simple workflow that fetches data from an
external API using the built-in `@blokjs/api-call` node and returns
the response.

## Prerequisites

- A Blok project with the HTTP trigger
- Node.js v22 or later

If you haven't created a project yet:

```bash
npx blokctl@latest create project
```

When prompted:
- Pick a project name
- Select **HTTP** as the trigger
- Select **NodeJS** as the runtime
- Choose **YES** when asked to install examples

## Workflow

Save this as `src/workflows/countries.ts`:

```ts src/workflows/countries.ts
import { workflow, step, http } from "@blokjs/core";
import apiCall from "@blokjs/api-call";
import { RespondNode } from "@blokjs/helpers";

export default workflow(
  "countries.list",
  { version: "1.0.0", trigger: http.get("/countries") },
  (req) => {
    // step(id, node, inputs) runs the node and returns a TYPED handle.
    const countries = step("fetch", apiCall, {
      url: "https://countriesnow.space/api/v0.1/countries/capital",
      method: "GET",
      headers: { "Content-Type": "application/json" },
      responseType: "application/json",
    });

    // Pass the handle straight into the next step — the runner resolves it.
    step("respond", RespondNode, { body: countries }, { ephemeral: true });
  },
);
```

The `name` follows the dotted `domain.action` convention so the
[typed client](/d/introduction/workflows#naming-convention) exposes it
as `blok.countries.list(...)`. Names must be unique across the
project — the registry rejects duplicates at boot.

This workflow:

<Steps>
  <Step title="Triggered by GET /countries">
    The `http.get("/countries")` trigger maps the route to this
    workflow.
  </Step>
  <Step title="Calls the external API">
    The `fetch` step runs the imported `apiCall` node to GET the
    country list. Its output auto-stores at `ctx.state.fetch` and is
    returned as the `countries` handle.
  </Step>
  <Step title="Returns the response">
    The `respond` step takes the `countries` handle as its body. It's
    `ephemeral` — it shapes the response without cluttering state.
  </Step>
</Steps>

## Customize

### Change the endpoint

Point the `url` at a different API:

```ts
const data = step("fetch", apiCall, {
  url: "https://your-api-endpoint.com/data",
  method: "GET",
});
```

### Add query parameters

```ts
const europe = step("fetch", apiCall, {
  url: "https://countriesnow.space/api/v0.1/countries/capital?filter=region&value=Europe",
  method: "GET",
});
```

### Add authentication

```ts
const data = step("fetch", apiCall, {
  url: "https://api.example.com/data",
  method: "GET",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer your-api-key",
  },
});
```

### Cache the result

Pass `idempotencyKey` in the 4th argument to make repeat requests free:

```ts
const countries = step(
  "fetch",
  apiCall,
  { url: "https://countriesnow.space/api/v0.1/countries/capital" },
  { idempotencyKey: "countries-list", idempotencyKeyTTL: 86_400_000 },
);
```

The first request hits the upstream API. Every subsequent request
within 24 hours returns the cached response without an HTTP call. See
[idempotent step caching](/d/reliability/idempotency).

### Transform the data

Add a step between `fetch` and `respond` to reshape the result. The
`ExprNode` helper (`@blokjs/expr`) evaluates a plain JS expression
against `ctx` and returns it — read the `countries` handle inside:

```ts
import { ExprNode } from "@blokjs/helpers";

const filtered = step("filter", ExprNode, {
  expression: "ctx.state.fetch.data.filter(c => c.region === 'Europe')",
});

step("respond", RespondNode, { body: filtered }, { ephemeral: true });
```
