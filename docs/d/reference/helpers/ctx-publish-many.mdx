Like [`@blokjs/ctx-publish`](./ctx-publish) but writes N keys in one
step. Returns `{ count }` of keys published.

## Quickstart

```ts
import { workflow, step, http } from "@blokjs/core";
import apiCall from "@blokjs/api-call";
import { CtxPublishManyNode } from "@blokjs/helpers";

export default workflow(
  "advance",
  { version: "1.0.0", trigger: http.get("/advance") },
  () => {
    const check = step("check", apiCall, { url: "https://example.com/status" });
    step("advance", CtxPublishManyNode, {
      values: {
        lastStatus: check.status,
        checkedAt: Date.now(),
      },
    });
  },
);
```

Each entry in `values` writes one key to `ctx.state` in a single step.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `values` | `Record<string, unknown>` | Object of key → value pairs to publish. Each entry sets `ctx.state[key] = value`. |

## Outputs

| Field | Notes |
|---|---|
| `count` | Number of keys published. |

## When to use

- Multiple computed fields from one logical step
- Replacing several adjacent `ctx-publish` steps with one
- Inside a loop body when each iteration updates several pieces of
  state
