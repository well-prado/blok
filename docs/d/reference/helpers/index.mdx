`@blokjs/helpers` ships 24 pre-built nodes covering the most common
workflow plumbing — logging, conditionals, KV state, HMAC verification,
JWT auth, LLM streaming + tool calls, HTTP response shaping, webhook
dispatch, and SSE / WS / worker / pub-sub publishers. Each helper has a
canonical `@blokjs/<name>` identifier.

In the handle DSL, import each helper as a **node object** by its
PascalCase name (`RespondNode`, `ExprNode`, …) and pass it to `step()`:

```ts
import { workflow, step, http } from "@blokjs/core";
import { ExprNode } from "@blokjs/helpers";

export default workflow("hello", { version: "1.0.0", trigger: http.get("/hello") }, () => {
  step("respond", ExprNode, { expression: "({ message: 'hello world' })" });
});
```

Helpers are auto-registered when `@blokjs/helpers` is in your project
deps. The `HELPER_NODES` barrel is spread into the runner's node
registry at boot, so the canonical `@blokjs/<name>` refs also resolve in
JSON workflows.

## Public surface & imports

Author workflows with `@blokjs/core` — `workflow`, `step`, `branch`,
`switchOn`, `tryCatch`, `forEach`, `loop`, the trigger helpers (`http`,
…), the typed comparators (`eq`, `ne`, `gt`, …), and `tpl`. Import the
helper nodes themselves as objects from `@blokjs/helpers`.

```ts
import { workflow, step, branch, tpl, http, eq } from "@blokjs/core";
import { RespondNode, ExprNode } from "@blokjs/helpers";
```

## Conventions

Every helper follows the same shape:

- **node object** — import its PascalCase export from `@blokjs/helpers`
  and pass it as `step()`'s second argument (the canonical `@blokjs/<name>`
  ref is what JSON workflows reference via `use:`)
- **`inputs`** — Zod-validated input fields, the third `step()` argument
  (see each helper page)
- Output stored at `ctx.state[<step-id>]` by default — opt out with
  `ephemeral: true`, rename with `as:`, flatten with `spread:` (the step
  options object, `step()`'s fourth argument)
- Errors surface as `GlobalError` with `code` propagated to the trigger's
  HTTP response (so `@blokjs/throw` with `code: 401` gives an actual 401)

## Composition

Helpers compose with the
[reliability knobs](/d/reliability/overview)
(`idempotencyKey`, `retry`, `maxDuration`) and
[primitives](/d/primitives/index) (`forEach`, `tryCatch`, etc.) like any
other step. An `@blokjs/api-call` wrapped in `idempotencyKey` + `retry`
inside a parallel `forEach` is the canonical fan-out shape.
