Validates `data` against a JSON Schema (draft-07) using Ajv. Throws a
`GlobalError` listing every validation error path on failure.

The alternative — Zod inside a custom `defineNode` — is fine too. Use
this when you want to keep schemas in JSON (e.g. shared across
languages, or generated from an OpenAPI spec).

Registered as `@blokjs/json-schema`. In the handle DSL, import the node
object (`JsonSchemaNode`) from `@blokjs/helpers` and pass it to `step()`.

## Quickstart

```ts
import { workflow, step, http } from "@blokjs/core";
import { JsonSchemaNode } from "@blokjs/helpers";

export default workflow("create-user", { version: "1.0.0", trigger: http.post("/users") }, (req) => {
  step("validate", JsonSchemaNode, {
    schema: {
      type: "object",
      required: ["name", "email"],
      properties: {
        name: { type: "string", minLength: 1 },
        email: { type: "string", format: "email" },
        age: { type: "integer", minimum: 0 },
      },
    },
    data: req.body,
  });
});
```

## Inputs

| Field | Type | Notes |
|---|---|---|
| `schema` | JSON Schema | Draft-07 schema object. Required. |
| `data` | unknown | The value to validate. Required. |

## Outputs

| Field | Notes |
|---|---|
| `valid` | `true` on success. |

On failure, throws `GlobalError` with `code: 400` and a body listing
every error path: `body.email format email; body.age must be >= 0`.

## Common patterns

### Request-body validation

```ts
import { workflow, step, http } from "@blokjs/core";
import { JsonSchemaNode } from "@blokjs/helpers";
import createUser from "../nodes/create-user";

export default workflow("create-user", { version: "1.0.0", trigger: http.post("/users") }, (req) => {
  step("validate", JsonSchemaNode, { schema: { /* ... */ }, data: req.body });
  step("create", createUser, { user: req.body });
});
```

The validate step throws 400 before any side-effects fire. The
remaining workflow only runs on valid input.

### Inside a tryCatch for graceful 400 responses

`ExprNode`'s `expression` is a raw `ctx.*` string, so read the error off
`ctx.error` directly:

```ts
import { tryCatch } from "@blokjs/core";
import { ExprNode, JsonSchemaNode } from "@blokjs/helpers";

tryCatch("validate-and-create", {
  try: () => {
    step("validate", JsonSchemaNode, { schema: { /* ... */ }, data: req.body });
    step("create", createUser, { user: req.body });
  },
  catch: () => {
    step("respond-error", ExprNode, {
      expression: "({ ok: false, validationErrors: ctx.error.message })",
    });
  },
});
```
