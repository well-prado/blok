Creates a subscription to one or more bus channels. The subscription
becomes the `source` for an [`@blokjs/sse-stream`](./sse-stream) step
that pumps events out to the connected browser.

Used inside [SSE-triggered workflows](/d/triggers/sse) — the
two-step pattern (subscribe → stream) is the canonical SSE shape.

## Quickstart

```ts
import { workflow, step } from "@blokjs/core";
import { SseSubscribeNode, SseStreamNode } from "@blokjs/helpers";

export default workflow(
  "chat.stream",
  {
    version: "1.0.0",
    // sse is a raw trigger block — pass it through opts.trigger.
    trigger: { sse: { path: "/sse/chat/:sessionId" } },
  },
  () => {
    const sub = step("sub", SseSubscribeNode, { channels: ["chat:{sessionId}"] });
    step("stream", SseStreamNode, { source: sub });
  },
);
```

`{sessionId}` resolves from `ctx.request.params.sessionId` at run time.
The `sub` handle flows straight into the `stream` step's `source` — no
`ctx.state` strings.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `channels` | `string[]` | Channel names. `{paramName}` placeholders interpolate from `ctx.request.params`. |

## Outputs

A subscription handle — opaque, used as the `source:` input on the
next `@blokjs/sse-stream` step. Don't inspect directly.

## Common patterns

### Per-session subscription

The [quickstart](#quickstart) above is exactly this — the `{sessionId}`
placeholder fills from the path param, so every connected browser
subscribes only to its own conversation.

### Multi-channel subscription

```ts
const sub = step("sub", SseSubscribeNode, {
  channels: ["feed", "alerts", "system"],
});
step("stream", SseStreamNode, { source: sub });
```

Events from any of the three channels stream out the one connection.
