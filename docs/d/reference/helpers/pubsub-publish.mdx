Producer-side companion of the [pub/sub trigger](/d/triggers/pubsub).
Publishes a message to a topic via the configured cloud provider.

Shares the adapter pool with the pub/sub trigger when both are present
in the same process — same connection, same credentials.

## Quickstart

```ts
import { workflow, step, http } from "@blokjs/core";
import postgresQuery from "../nodes/postgres-query";
import { PubsubPublishNode } from "@blokjs/helpers";

export default workflow(
  "create-order",
  { version: "1.0.0", trigger: http.post("/orders") },
  (req) => {
    const create = step("create", postgresQuery, {
      query: "INSERT INTO orders ... RETURNING id, total",
      params: [req.body.items],
    });
    step("publish", PubsubPublishNode, {
      provider: "gcp",
      topic: "orders.placed",
      message: { orderId: create.id, total: create.total },
    });
  },
);
```

The `publish` step emits the event off the `create` handle — no
`ctx.state` strings.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `provider` | `"gcp"` \| `"aws"` \| `"azure"` | Cloud provider. Falls back to env var if omitted. |
| `topic` | `string` | Provider-specific identifier (GCP topic name / SNS ARN / Service Bus topic). |
| `message` | unknown | Payload. JSON-serialized on the wire. |
| `attributes` | `Record<string, string>` | Optional message attributes (provider-specific). |

## Outputs

| Field | Notes |
|---|---|
| `messageId` | Provider-issued message id. |
| `provider` | Provider that handled the publish. |

## Common patterns

### Decoupled event emission

The [quickstart](#quickstart) above is exactly this pattern — write to the
database, then emit the event off the returned handle. Downstream services
subscribe via [pub/sub trigger](/d/triggers/pubsub) on `orders.placed` and
react asynchronously.

### Provider-agnostic via env var

```ts
step("publish", PubsubPublishNode, {
  topic: "orders.placed",
  message: create.data,
});
```

Without an explicit `provider`, the helper reads
`BLOK_PUBSUB_PROVIDER` from env. Switch clouds without code changes.
