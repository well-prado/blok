Closes the WS connection that triggered the workflow. Sends the
standard WebSocket close frame with the supplied code + reason.

## Quickstart

```ts
import { workflow, step } from "@blokjs/core";
import { WsCloseNode } from "@blokjs/helpers";

export default workflow(
  "ws-guard",
  {
    version: "1.0.0",
    // websocket is a raw trigger block — pass it through opts.trigger.
    trigger: { websocket: { path: "/ws/secure", events: ["open"] } },
  },
  () => {
    step("close", WsCloseNode, { code: 4001, reason: "Invalid token" });
  },
);
```

## Inputs

| Field | Type | Default | Notes |
|---|---|---|---|
| `code` | `number` | `1000` | WebSocket close code. `1000` = normal closure. `4000-4999` are app-defined. |
| `reason` | `string` | `""` | Human-readable reason. Surfaced to the client via `event.reason`. |

## Outputs

`{ closed: true, code, reason }`.

## Common patterns

### Auth-fail close

`branch` conditions are evaluated as raw `ctx.*` expressions (never
`$.`/`js/`-prefixed), so write them as plain strings:

```ts
import { workflow, step, branch } from "@blokjs/core";
import { WsCloseNode } from "@blokjs/helpers";

export default workflow(
  "ws-auth-check",
  {
    version: "1.0.0",
    trigger: { websocket: { path: "/ws/secure", events: ["open", "message"] } },
  },
  () => {
    branch("verify", "ctx.request.headers['authorization'] === undefined", {
      then: () => {
        step("close", WsCloseNode, { code: 4001, reason: "missing auth header" });
      },
    });
  },
);
```

### Idle timeout

Trigger on `events: ["message"]`, track last-activity in a KV, close
when stale:

```ts
branch("close-if-idle", "(Date.now() - (ctx.state.lastActivity || 0)) > 300000", {
  then: () => {
    step("close", WsCloseNode, { code: 1000, reason: "idle timeout" });
  },
});
```

## Close code reference

| Code | Meaning |
|---|---|
| `1000` | Normal closure. |
| `1001` | Going away (server shutting down). |
| `1002` | Protocol error. |
| `1008` | Policy violation (use for auth failures with informational reason). |
| `1011` | Server error. |
| `4000-4999` | App-defined. Document your codes in workflow comments. |
