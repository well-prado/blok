Broadcasts a message to all WS connections matching a path / channel.
In-process by default — every connection on the current Node process
receives. With `BLOK_WS_BACKPLANE=redis`, broadcasts cross processes
via Redis pub/sub.

## Quickstart

```ts
import { workflow, step, tpl } from "@blokjs/core";
import { WsBroadcastNode } from "@blokjs/helpers";

export default workflow(
  "ws-room",
  {
    version: "1.0.0",
    // websocket is a raw trigger block — pass it through opts.trigger.
    trigger: { websocket: { path: "/ws/room/:roomId", events: ["message"] } },
  },
  (conn) => {
    step("broadcast", WsBroadcastNode, {
      channel: tpl`room-${conn.params.roomId}`,
      message: conn.body,
    });
  },
);
```

Every WS connection in the room sees the message. `tpl` embeds the
`roomId` path handle into the channel string.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `channel` | `string` | Channel / room identifier. WS connections on the same channel all receive. |
| `message` | unknown | Payload (JSON-serialized). |

## Outputs

| Field | Notes |
|---|---|
| `delivered` | Number of connections that received the broadcast (local count when no backplane; total when backplane is set). |

## Cross-process broadcast (backplane)

Default: in-process only. Set:

```bash
BLOK_WS_BACKPLANE=redis
BLOK_WS_BACKPLANE_REDIS_URL=redis://...
```

The backplane fans broadcasts through Redis pub/sub so every process
delivers to its local connections. Per-connection
[`ws-reply`](./ws-reply) doesn't need the backplane.

## Common patterns

### Chat room

The [quickstart](#quickstart) above is exactly the chat-room shape — a
`websocket` trigger on `/ws/room/:roomId` broadcasting each inbound
message to everyone in that room.

### Server-pushed event from HTTP

```ts
import { workflow, step, http } from "@blokjs/core";
import { WsBroadcastNode } from "@blokjs/helpers";

export default workflow(
  "push-update",
  { version: "1.0.0", trigger: http.post("/push") },
  (req) => {
    step("push", WsBroadcastNode, { channel: "live", message: req.body });
  },
);
```

Connected browsers on `/ws/live` receive every POST without an
intermediate event bus.
