Appends an event to a 1000-entry in-process ring buffer for inspection
via Studio or the `getAuditEvents()` test utility. The ring is
bounded — oldest entries drop when full.

For production audit trails, write to a durable store (Postgres, S3,
Datadog) instead. This helper is the dev / test stand-in.

Registered as `@blokjs/audit-log`. In the handle DSL, import the node
object (`AuditLogNode`) from `@blokjs/helpers` and pass it to `step()`.

## Quickstart

```ts
import { workflow, step, http } from "@blokjs/core";
import { AuditLogNode, JwtVerifyNode } from "@blokjs/helpers";

export default workflow("delete-user", { version: "1.0.0", trigger: http.delete("/users/:userId") }, (req) => {
  const auth = step("auth", JwtVerifyNode, { token: "...", secret: process.env.JWT_SECRET });

  step("audit", AuditLogNode, {
    event: "user.deleted",
    attrs: { userId: req.params.userId, deletedBy: auth.subject },
  }, { ephemeral: true });
});
```

## Inputs

| Field | Type | Notes |
|---|---|---|
| `event` | `string` | Event name. Required. |
| `attrs` | `Record<string, unknown>` | Optional event metadata. |

## Outputs

| Field | Notes |
|---|---|
| `event` | Echo of the event name. |
| `attrs` | Echo of attrs. |
| `timestamp` | Server-side timestamp (unix ms). |
| `requestId` | `ctx.id` of the current run. |

## Reading the audit ring

```typescript
import { getAuditEvents } from "@blokjs/helpers";

const events = getAuditEvents();
// → [{ event: "user.deleted", attrs: {...}, timestamp, requestId }, ...]
```

The ring is bounded at 1000 entries; older entries drop.

## Production replacement

The recommended production shape: write the same audit fields to a
durable store via an `@blokjs/api-call` node or a custom node:

```ts
import ApiCallNode from "@blokjs/api-call";

step("audit-prod", ApiCallNode, {
  url: "https://audit-service.internal/events",
  method: "POST",
  body: { event: "user.deleted", userId: req.params.userId, ts: Date.now() },
}, { ephemeral: true });
```
