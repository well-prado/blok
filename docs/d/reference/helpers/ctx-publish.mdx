v2 default-stores every step's output to `ctx.state[step.id]`. Use
`@blokjs/ctx-publish` when you want to publish under a name that isn't
the step id — typically inside a loop / branch arm to accumulate state
across iterations.

## Quickstart

```ts
import { workflow, step, http } from "@blokjs/core";
import apiCall from "@blokjs/api-call";
import { CtxPublishNode } from "@blokjs/helpers";

export default workflow(
  "accumulate",
  { version: "1.0.0", trigger: http.get("/totals") },
  () => {
    const fetch = step("fetch", apiCall, { url: "https://example.com/api" });
    step("accumulate", CtxPublishNode, {
      name: "totals",
      value: fetch.data.amount,
    });
  },
);
```

The `value` is computed from the `fetch` handle and published under the
explicit name `totals` instead of the step id.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `name` | `string` | Key to write under `ctx.state`. Required. |
| `value` | unknown | Value to publish. |

## Outputs

`{ name, value }` echo.

`ctx.state[name]` AND `ctx.vars[name]` (legacy alias) are both
written.

## Common patterns

### Loop accumulator

`loop` is a JSON-only step shape (not part of the typed-handle DSL — see
[loop](/d/primitives/loop)), so its body uses raw `ctx.*` expressions:

```json
"loop": {
  "until": "ctx.state['fetch'].data.nextCursor === null",
  "maxIterations": 100,
  "do": [
    { "id": "fetch", "use": "@blokjs/api-call", "inputs": { "url": "..." } },
    {
      "id": "accumulate",
      "use": "@blokjs/ctx-publish",
      "inputs": {
        "name": "results",
        "value": "[...(ctx.state.results || []), ...ctx.state.fetch.data.items]"
      }
    }
  ]
}
```

The loop body's `fetch` step's data is per-iteration; `results`
accumulates across iterations under a stable name.

### Stash auth identity from a verify step

```ts
import { workflow, step, http } from "@blokjs/core";
import { JwtVerifyNode, CtxPublishNode } from "@blokjs/helpers";

export default workflow(
  "stash-identity",
  { version: "1.0.0", trigger: http.get("/me") },
  (req) => {
    const verify = step("verify", JwtVerifyNode, { token: req.headers.authorization });
    step("stash-identity", CtxPublishNode, {
      name: "identity",
      value: { userId: verify.subject, claims: verify.claims },
    });
  },
);
```

Downstream steps read `ctx.state.identity.userId` without coupling to
the verify step's id.

## Composition

- For multiple keys in one step, use [`ctx-publish-many`](./ctx-publish-many)
- For computed values, prefer [`expr`](./expr) — its return goes to
  `ctx.state[<step-id>]` by default, no rename needed
- Pair with `ephemeral: true` to avoid the redundant
  `ctx.state[<step-id>]` entry (only the published `name` survives)
