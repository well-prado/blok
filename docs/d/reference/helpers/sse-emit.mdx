Writes a **single** SSE frame to the connected client and returns
`{ sent: true }`. Unlike [`@blokjs/sse-stream`](./sse-stream) (which blocks,
pumping a subscription), `sse-emit` is a one-shot ŌĆö use it to punctuate a
workflow with discrete control frames (`message_ids`, `workflow_start`,
`complete`, `error`) between ordinary steps, without the
publishŌåÆsubscribeŌåÆpump dance.

Use only inside [SSE-triggered](/d/triggers/sse) workflows; `ctx.stream` is
undefined elsewhere and the step throws.

## Quickstart

```ts
import { workflow, step } from "@blokjs/core";
import { SseEmitNode } from "@blokjs/helpers";

export default workflow(
  "agent.stream",
  {
    version: "1.0.0",
    // sse is a raw trigger block ŌĆö pass it through opts.trigger.
    trigger: { sse: { path: "/sse/agent/:sessionId" } },
  },
  () => {
    step(
      "open",
      SseEmitNode,
      { event: "workflow_start", data: { ok: true } },
      { ephemeral: true },
    );
    // ŌĆ” long-running streaming node ŌĆ”
  },
);
```

`sse-emit` works only under an `sse` trigger ŌĆö `ctx.stream` is undefined
elsewhere and the step throws. Pass `ephemeral: true` (4th `step` arg) so
the control frame doesn't clutter `ctx.state`.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `event` | `string` | SSE event name ŌåÆ the frame's `event:` field. Optional. |
| `data` | `unknown` | Event payload. JSON-serialized when not a string. Required. |
| `id` | `string` | Frame `id:` ŌĆö clients echo it as `Last-Event-Id` on reconnect. Optional. |
| `retry` | `number` | Reconnection hint in ms ŌåÆ the SSE `retry:` field. Optional. |

## Behavior

- Emits one `event: <name>\ndata: <json>\n\n` frame via `ctx.stream.writeSSE(...)`.
- Returns `{ sent: true }`; pair with `ephemeral: true` so it doesn't store state.
- Throws if `ctx.stream` is absent (i.e. not under `trigger.sse`).

## Where it fits

The canonical streaming-agent shape emits a fixed event vocabulary around a
long-running node: `message_ids ŌåÆ workflow_start ŌåÆ ŌĆ” live frames ŌĆ” ŌåÆ
complete`. `sse-emit` writes the bookend control frames; the live middle is
produced by a streaming node (see [`@blokjs/sse-stream`](./sse-stream) for the
bus-pump path, or a `runtime.*` node with `streamTo: "sse"` for native
runtimeŌåÆSSE forwarding).
