Logs a message via the runner's `ctx.logger.logLevel(level, message)`.
Errors route to `ctx.logger.error(message, stack)` so the trace store
sees the proper severity badge.

Registered as `@blokjs/log`. In the handle DSL, import the node object
(`LogNode`) from `@blokjs/helpers` and pass it to `step()`.

## Quickstart

```ts
import { workflow, step, tpl, http } from "@blokjs/core";
import { LogNode } from "@blokjs/helpers";

export default workflow("process-items", { version: "1.0.0", trigger: http.post("/process") }, () => {
  const fetch = step("fetch", fetchItems, {});

  step("log", LogNode, {
    level: "info",
    message: tpl`processed ${fetch.data.length} items`,
  }, { ephemeral: true });
});
```

`ephemeral: true` is the typical pattern — the log is a side effect,
not state for downstream steps. `tpl` interpolates handles into the
message string.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `level` | `"info"` \| `"warn"` \| `"error"` \| `"debug"` | Log level. |
| `message` | `string` | The log line. |
| `attrs` | `Record<string, unknown>` (optional) | Extra fields appended as JSON to the line. |

## Outputs

| Field | Notes |
|---|---|
| `level` | Echo of the input level. |
| `message` | Echo of the input message. |

(Mainly for chaining and Studio inspection. Use `ephemeral: true` if
you don't need state propagation.)

## Where the log goes

`ctx.logger` is the trigger's logger context — typically wired to:

- **Console** when `CONSOLE_LOG_ACTIVE=true` (development default)
- **The trace store** so the log appears in the run detail in Studio
- **External log aggregators** if you've configured an OpenTelemetry
  log exporter

## Common patterns

### Step-trace breadcrumbs

```ts
const lookup = step("lookup", lookupUser, { id: req.params.id });

step("log-progress", LogNode, {
  level: "info",
  message: tpl`fetched user ${lookup.data.id}`,
  attrs: { userId: lookup.data.id },
}, { ephemeral: true });
```

The `attrs` show up as a JSON suffix on the log line — searchable in
Studio's log filter.

### Conditional warning

```ts
import { branch, eq } from "@blokjs/core";

branch("maybe-warn", eq(fetch.data.length, 0), {
  then: () => {
    step("warn", LogNode, { level: "warn", message: "empty result set" }, { ephemeral: true });
  },
});
```
