Sends a message back to the WS connection that fired the workflow.
Used in [WebSocket workflows](/d/triggers/websocket) for request-style
patterns (echo, RPC, per-client updates).

## Quickstart

```ts
import { workflow, step } from "@blokjs/core";
import { WsReplyNode } from "@blokjs/helpers";

export default workflow(
  "ws-echo",
  {
    version: "1.0.0",
    // websocket is a raw trigger block — pass it through opts.trigger.
    trigger: { websocket: { path: "/ws/echo", events: ["message"] } },
  },
  (conn) => {
    step("reply", WsReplyNode, {
      message: { echo: conn.body, at: Date.now() },
    });
  },
);
```

`ws-reply` writes back to the exact connection that fired the run.

## Inputs

| Field | Type | Notes |
|---|---|---|
| `message` | unknown | The payload to send. Serialized to JSON on the wire (string messages pass through verbatim). |

## Outputs

`{ sent: true }`.

## Behavior

The trigger tracks the connection that fired the workflow via
`ctx.vars._ws_connection`. `@blokjs/ws-reply` writes to that connection
directly — no need to specify a connection id.

For server-pushed events to ALL connections, see
[`@blokjs/ws-broadcast`](./ws-broadcast). For closing the connection,
see [`@blokjs/ws-close`](./ws-close).
