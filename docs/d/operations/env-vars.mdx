## Tracing & storage

Controls the run store backing every other feature on this page â€”
disabling tracing also disables the idempotency cache, the
concurrency gate, and the deferred scheduler (they all share the
store).

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_TRACE_ENABLED` | `true` | Master kill-switch. `false` disables tracing AND every store-backed feature (cache, gate, scheduler). |
| `BLOK_TRACE_STORE` | `sqlite` | Run store backend: `sqlite`, `inmemory`, or `postgres`. |
| `BLOK_TRACE_SQLITE_PATH` | `.blok/runs.db` | SQLite database path when `BLOK_TRACE_STORE=sqlite`. |
| `BLOK_TRACE_PG_*` | unset | Postgres connection params (host, port, user, password, database). See [trace storage Â· Postgres](/d/reference/trace-storage#postgres-store). |
| `BLOK_TRACE_RETENTION_DAYS` | `30` | Older runs evicted by `evictOldRuns()`. The [janitor](/d/operations/janitor-and-shutdown#janitor) handles ancillary tables (cache, locks, dispatches). |
| `BLOK_INDEXED_METADATA_KEYS` | unset | Comma-separated metadata keys to promote to indexed columns on `workflow_runs` (F1). Each declared key gets a `metadata_<key>_idx` VIRTUAL generated column + btree index. Queries against declared keys use the index instead of a sequential `json_extract` scan â€” meaningful for 100K+ row stores. Example: `BLOK_INDEXED_METADATA_KEYS=tier,region`. Keys outside `^[a-zA-Z0-9_-]+$` are silently dropped. SQLite-only â€” the in-memory and Postgres-via-mirror stores already keep everything in memory. |

## Workflow loading

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_FILE_BASED_ROUTING` | `true` | **Default ON since v0.6.** Auto-discover JSON workflows under `workflows/json/` and register each at its `trigger.http.path`. Set to `false` to fall back to the legacy `/<workflow-key>/<sub>` catch-all dispatch (deprecated, will be removed). |
| `BLOK_ROUTING_LEGACY` | unset | Full legacy escape hatch. When `=1` / `=true`, file-based routing is disabled AND workflows without an explicit `trigger.http.path` get the filename-derived URL instead of throwing `MissingExplicitPathError`. Will be removed in a future release. |
| `BLOK_HMR` | `true` (dev) | Hot-reload workflows on file change. Triggers re-scan and `WorkflowRegistry.clear()` + `registerAll()`. |
| `BLOK_MAPPER_MODE` | `strict` | Input expression resolution failure mode. `strict` (default) throws `MapperResolutionError` on a failed expression, `warn` logs + passes the literal through (old v1 behavior), `silent` suppresses. See [debugging workflows](/d/introduction/workflows#debugging). |

## Sub-workflows

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_MAX_SUBWORKFLOW_DEPTH` | `10` | Recursion cap for sub-workflow steps. Throws `Sub-workflow recursion limit exceeded` when exceeded. See [sub-workflows Â· recursion guard](/d/composition/sub-workflows#recursion-guard). |
| `BLOK_SELF_BASE_URL` | `http://localhost:${PORT \|\| 4000}` | G2 (v0.6) â€” base URL used by `dispatch: "http-self"` sub-workflow steps when making the self-call. Set to the deployment's externally-reachable service URL in Docker/k8s envs (e.g. `http://blok-svc:4000`); the default `localhost` only works for single-process dev. Ignored for `dispatch: "in-process"` (the default). |

## Node env-var allowlist

Operator-controlled gate on which environment variables nodes can read
through `ctx.env`. Default behaviour (neither set) is allow-all â€”
preserves backwards compatibility. Setting either makes `ctx.env` a
filtering Proxy. See [threat model Â· environment variables and node trust](/d/security/threat-model#environment-variables-and-node-trust).

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_NODE_ENV_ALLOW` | unset | Comma-separated exact names. Example: `DATABASE_URL,STRIPE_KEY`. |
| `BLOK_NODE_ENV_ALLOW_PREFIX` | unset | Comma-separated prefixes. Example: `PUBLIC_,SAFE_`. |
| `BLOK_SUPPRESS_ENV_ALLOW_WARNING` | unset | Set to `1` to silence the boot warning when `BLOK_ENV=production` and no allowlist is configured. |

## Concurrency gate

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_CONCURRENCY_DISABLED` | unset | Set to `1` to short-circuit the [concurrency gate](/d/reliability/concurrency-keys) entirely. Runs proceed regardless of in-flight count. |
| `BLOK_CONCURRENCY_LEASE_MS` | `3600000` | Process-wide override for `concurrencyLeaseMs`. Per-trigger override takes precedence. |

## Scheduling gates

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_SCHEDULING_DISABLED` | unset | Set to `1` to short-circuit `delay`, `ttl`, AND `debounce` gates. Runs proceed synchronously. See [scheduling overview](/d/scheduling/overview). |
| `BLOK_SCHEDULER_CLAIM_DISABLED` | unset | Set to `1` to disable Tier C #2 claim-based recovery â€” `HttpTrigger.recoverDispatches()` reverts to the legacy `getScheduledDispatches` path. Use only if you've verified your deployment doesn't share a PG store across processes. |
| `BLOK_SCHEDULER_CLAIM_LEASE_MS` | `60000` | Tier C #2 â€” cross-process scheduler claim lease. Recovered dispatches are claimed by the recovering process; another process can take over a claim older than this. Tune for trade-off between handoff latency (low) and false-takeover risk (high). |
| `BLOK_SCHEDULER_HEARTBEAT_INTERVAL_MS` | `20000` | Tier C #2 â€” how often the `DeferredRunScheduler` refreshes `claimed_at` for active claims. Should be ~3Ă— smaller than the claim lease so a dead process's claims expire before peers spuriously take over. |

## Crash recovery

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_CRASH_AUTOFLIP_DISABLED` | unset | Set to `1` to skip `installCrashHandlers()` (no auto-flip on `uncaughtException` / `unhandledRejection`). Useful if you have your own crash handling. See [crash auto-flip](/d/reliability/cancellation#crash-auto-flip). |
| `BLOK_ORPHAN_THRESHOLD_MS` | `120000` | Threshold for [orphan recovery](/d/reliability/cancellation#orphan-recovery) at boot. Runs in `running` status with `startedAt` older than this are flipped to `crashed`. |

## Janitor

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_JANITOR_DISABLED` | unset | Set to `1` to skip the periodic storage [janitor](/d/operations/janitor-and-shutdown#janitor). |
| `BLOK_JANITOR_INTERVAL_MS` | `300000` | Sweep interval. Sweeps `idempotency_cache`, `concurrency_locks`, and `scheduled_dispatches` for expired rows. |

## Graceful shutdown

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_GRACEFUL_SHUTDOWN_DISABLED` | unset | Set to `1` to skip `installShutdownHandlers()` (no SIGTERM/SIGINT drain). The process exits immediately on signal, leaving in-flight work to be picked up by [orphan recovery](/d/reliability/cancellation#orphan-recovery) on the next boot. See [graceful shutdown](/d/operations/janitor-and-shutdown#graceful-shutdown). |

## Logging

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_STREAM_LOGS` | unset | Enable live log streaming for all steps. Per-step opt-in via `stream_logs: true` overrides. |

## Metrics

| Variable | Default | Purpose |
|---|---|---|
| `BLOK_METRICS_DISABLED` | unset | Opt **out** of metrics. The `~33 blok_*` Prometheus series and the `/metrics` exporter are **on by default** for backward compatibility; set this to `1` (or `true`) to skip the OTel meter provider and the exporter entirely (no import-time side effect). There is **no** `BLOK_METRICS_ENABLED`. Managed per-project by [`blokctl observability`](/d/cli/observability). |
| `BLOK_METRICS_PER_KEY` | unset | D6 â€” opt in to per-`concurrency_key` labels on the `blok_concurrency_*` OTel counters and `blok_concurrency_occ_retries` histogram. Default OFF strips the key before emission so deployments with high-cardinality keys (per-user, per-tenant) don't blow up metrics cardinality. Set to `1` or `true` to keep the key as a label when you have a small, bounded key space and want per-bucket granularity for dashboards / alerts. |

## Observability modules

Environment for the opt-in [`blokctl observability`](/d/cli/observability)
modules. Tracing and the error sink are **inert** until you supply an
endpoint / DSN.

| Variable | Default | Purpose |
|---|---|---|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | unset | OTLP collector endpoint for the `tracing` module (e.g. `http://localhost:4318` â†’ Tempo/Jaeger). Spans are emitted only when this is set. |
| `BLOK_TRACING_DISABLED` | unset | Force-disable OTLP span export even when an endpoint is set. |
| `SENTRY_DSN` | unset | Destination for the `error-sink` module â€” unhandled errors are forwarded to this Sentry-compatible sink. Inert when unset. |

## Runtime adapters

Connecting non-TypeScript nodes (Python, Go, Rust, etc.) to the runner.

| Variable | Default | Purpose |
|---|---|---|
| `RUNTIME_TRANSPORT` | `grpc` | gRPC is the sole runtime transport since v0.5. Setting this to `http` (or any other value) throws at trigger boot â€” drop the env var. |
| `RUNTIME_<KIND>_TRANSPORT` | `grpc` | Per-kind override. Setting to `http` throws â€” same removal as above. |
| `RUNTIME_<KIND>_HOST` | `localhost` | Hostname for the SDK container per language. |
| `RUNTIME_<KIND>_GRPC_PORT` | varies | gRPC port per language. Conventionally `<httpPort> + 1000` â€” Go `10001`, Rust `10002`, Java `10003`, C# `10004`, PHP `10005`, Ruby `10006`, Python3 `10007`. |
| `RUNTIME_<KIND>_PORT` | varies | Legacy port field retained for back-compat with older configs; not health-probed by the CLI since v0.5. |

For C# specifically, env keys preserve dots (e.g.
`RUNTIME_CSHARP_HOST` becomes `Runtime.Csharp.Host` in the .NET
config). For Java, `NettyServerBuilder` keepalive tuning matters under
heavy concurrent load.

## Kill-switches

The full set of `*_DISABLED` variables, in one place. Each can be
toggled without redeploying code:

| Variable | Disables |
|---|---|
| `BLOK_TRACE_ENABLED=false` | Tracing AND every store-backed feature (cache, gate, scheduler). |
| `BLOK_CONCURRENCY_DISABLED=1` | The concurrency gate. Runs proceed regardless of in-flight count. |
| `BLOK_SCHEDULING_DISABLED=1` | The `delay` / `ttl` / `debounce` gates. Runs proceed synchronously. |
| `BLOK_CRASH_AUTOFLIP_DISABLED=1` | `uncaughtException` / `unhandledRejection` auto-flip to `crashed`. |
| `BLOK_JANITOR_DISABLED=1` | The periodic storage cleanup sweep. |
| `BLOK_GRACEFUL_SHUTDOWN_DISABLED=1` | The SIGTERM/SIGINT drain handler. |
| `BLOK_METRICS_DISABLED=1` | The Prometheus metrics + `/metrics` exporter (on by default). |

Use these for emergency operational recovery, not as long-term
configuration. The defaults are tuned for production correctness;
disabling them shifts work to operator vigilance.

## Migration & durability scope

A summary of which feature persists where, by store. See the linked
feature pages for full mechanics.

| Feature | SQLite (default) | InMemory | Postgres |
|---|---|---|---|
| Run + node trace | yes (durable) | process-local | yes (durable) |
| [Idempotency cache](/d/reliability/idempotency) | yes (migration v4) | process-local | yes (migration v3 â€” hybrid mirror) |
| [Concurrency locks](/d/reliability/concurrency-keys) | yes (migration v7) | process-local | yes (migration v3 â€” hybrid mirror) |
| [Scheduled dispatches](/d/scheduling/delay-and-ttl#durability) | yes (migration v9) | process-local | yes (migration v3 â€” hybrid mirror) |

Within-process gating works across all backends. Cross-restart
durability requires SQLite or Postgres.
