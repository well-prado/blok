## Operating mental model

Three layers of state determine production behavior:

1. **Hot path** — in-memory data structures: scheduler timers, debounce
   coordinator, in-process concurrency Map, idempotency cache mirror.
   Lost on process exit; reconstructed from the durable layer on boot.
2. **Durable layer** — sqlite (default) or Postgres. Stores
   `workflow_runs`, `node_runs`, `idempotency_cache`,
   `concurrency_locks`, and `scheduled_dispatches`. Survives restart.

Operators tune env vars against the layer they're targeting:

- **Hot path tuning** — `BLOK_*_DISABLED` kill-switches, lease
  durations, janitor intervals.
- **Durable layer tuning** — `BLOK_TRACE_STORE`, `BLOK_TRACE_PG_*`,
  `BLOK_TRACE_RETENTION_DAYS`.

The [env vars page](/d/operations/env-vars) groups them in this same
order so you can find what you're looking for by intent rather than
prefix.

## When to reach for what

<AccordionGroup>
  <Accordion title="Production traffic spike — need to relieve back-pressure">
    Set `BLOK_CONCURRENCY_DISABLED=1` or `BLOK_SCHEDULING_DISABLED=1`
    as a kill-switch (no redeploy). Both flip the runtime to
    "synchronous, no gating" — runs proceed without the gate but the
    rest of the pipeline still works. See
    [env vars · kill-switches](/d/operations/env-vars#kill-switches).
  </Accordion>

  <Accordion title="Need to know what's stuck in flight right now">
    `curl /__blok/concurrency/state` for live `(workflow, key)`
    buckets. The same data feeds Studio's
    [in-flight tile](/d/studio/index#in-flight-concurrency-tile).
  </Accordion>

  <Accordion title="Storage is growing unbounded">
    Tune the [janitor](/d/operations/janitor-and-shutdown#janitor)
    interval down. Default 5 minutes; aggressive deployments can run
    every 60 seconds.
  </Accordion>

  <Accordion title="Restart leaves runs stuck in `running`">
    [Orphan recovery](/d/reliability/cancellation#orphan-recovery)
    flips them to `crashed` after `BLOK_ORPHAN_THRESHOLD_MS` (default
    2 min). Tune up if your workflows legitimately run longer.
  </Accordion>

  <Accordion title="Adding a second Blok process behind a load balancer">
    The in-process concurrency gate can't coordinate across processes —
    each tracks its own slots (effective limit `concurrencyLimit × N`).
    Pin a key to one process or run one process per key space;
    cross-process coordination is not yet available. See
    [concurrency keys · multi-process scope](/d/reliability/concurrency-keys#multi-process-scope).
  </Accordion>

  <Accordion title="Need Prometheus metrics in our existing dashboard">
    Six new OpenTelemetry counters surface through the existing
    Prometheus exporter — see
    [observability](/d/operations/observability#opentelemetry-counters).
    No additional config needed if you already scrape Blok.
  </Accordion>
</AccordionGroup>
